## üöÄ Getting Started

### Prerequisites
- Basic knowledge of HTML, CSS, JavaScript
- Familiarity with REST APIs
- Understanding of responsive design principles

### Development Setup
1. Fork the repository
2. Clone your fork:
   ```bash
   git clone https://github.com/your-username/indian-weather-app.git
   cd indian-weather-app
   ```
3. Get API keys:
   - [OpenWeatherMap API key](https://openweathermap.org/api)
   - [NewsAPI key](https://newsapi.org/)
4. Update `script.js` with your API keys
5. Open `index.html` in your browser or use a local server

### Making Changes
1. Create a new branch:
   ```bash
   git checkout -b feature/your-feature-name
   ```
2. Make your changes
3. Test thoroughly across different devices and browsers
4. Commit your changes:
   ```bash
   git commit -m "Add: descriptive commit message"
   ```
5. Push to your fork:
   ```bash
   git push origin feature/your-feature-name
   ```
6. Create a Pull Request

## üìù Coding Guidelines

### JavaScript
- Use modern ES6+ features
- Add JSDoc comments for functions
- Handle errors gracefully
- Follow the existing code structure
- Use meaningful variable and function names

### CSS
- Use CSS custom properties (variables) defined in `:root`
- Follow BEM methodology for class naming
- Ensure responsive design (mobile-first approach)
- Use semantic class names
- Group related styles together

### HTML
- Use semantic HTML5 elements
- Include proper ARIA labels for accessibility
- Ensure proper heading hierarchy
- Add alt text for images

## üêõ Reporting Issues

When reporting issues, please include:
- Browser and version
- Device type (mobile/desktop)
- Steps to reproduce
- Expected behavior
- Actual behavior
- Screenshots (if applicable)

### Issue Labels
- `bug`: Something isn't working
- `enhancement`: New feature request
- `documentation`: Improvements to docs
- `good first issue`: Good for newcomers
- `help wanted`: Extra attention needed

## üîÑ Pull Request Process

1. **Description**: Provide a clear description of changes
2. **Testing**: Ensure all functionality works correctly
3. **Screenshots**: Include before/after screenshots for UI changes
4. **Documentation**: Update README if needed
5. **Code Review**: Address any review feedback promptly

### Pull Request Checklist
- [ ] Code follows the project's style guidelines
- [ ] Self-review completed
- [ ] Code is well-commented
- [ ] Changes tested on multiple browsers/devices
- [ ] Documentation updated (if applicable)
- [ ] No console errors or warnings
